한 곳에 깊게 박는 것보다 운영의 모든 곳에 얇게 깔리는 것이 임팩트가 큽니다. 합치면 직원 2~3명 분의 일을 합니다.

① 설계

쇼가든 설계 — 매번 손으로 하던 것을 시스템으로

쇼가든 = 박람회·전시장에 설치하는 작은 정원. 의뢰가 들어오면 매번 새로 설계.

  1. 1클라이언트 인터뷰대면·전화·카톡으로 산발적. 정리 안 됨.→ 카톡 자동 정리
  2. 2컨셉 발굴레퍼런스 사진 폴더 뒤짐. 매번 새로.→ AI 이미지 추천
  3. 3식재 계획머릿속 식물 DB. 협력사 재고 전화.→ 식물 DB + 재고 연동
  4. 4견적·도면엑셀·캐드 수작업. 견적 한 건당 반나절.→ 자동 견적 생성
  5. 5시공 일정달력 수동 기입. 협력사 일정 충돌.→ 일정 자동 통합
핵심은 끝판왕 자동화가 아니라, 각 단계에서 "내 머리에만 있던 것을 시스템 밖으로 꺼내는 것"입니다.

② 시공·관리

그린월 → 기억정원 (시니어 복지 사업)

그린월 = 카페·로비 벽면에 식물을 빽빽하게 심는 벽면 녹화. 기억정원 = 어르신 인지 회복 프로그램.

피벗 스토리

처음엔 "어반정글 그린월 관리 앱"으로 시작. 운영하다 보니 고객의 70%가 시니어 복지관임을 발견. 그래서 제품을 "기억정원 — 어르신 인지 회복 + 디지털 적응"으로 방향 전환.

도메인 깊이가 없었다면 못 본 신호. AI로 만든 시스템이 가벼워서 2주 안에 방향 전환 가능했음.

실제 기능

어르신: 주간 체크인 / 옛 사진 회상 / 키오스크·스마트폰 연습 / 만족도 표시

관리자: 이용자 관리 / 운영 인사이트 / 직원 개입 기록 / 공간 분석(동선) / 환경 모니터링

③ 영업·자금

지원사업 레이더 — 사업 자금을 놓치지 않는 시스템

왜 만들었나

농업·조경·치유산업 정부 지원사업은 매년 수백 건인데 사이트가 흩어져 있고 양식이 다 다릅니다. 운영자로서 매번 놓쳤습니다. 한 건 놓치면 한 해 사업 방향이 바뀔 만큼 큽니다.

구조

서버 맥에 24시간 가동. 28개 지원사업 사이트 자동 모니터링 + AI가 적합도 점수 매김.

⑤ 컨설팅

대영케이블 컨설팅 — AI를 매출로 바꾸기

프로젝트명 "살핌" — 나·일·상대를 살피는 마음. 대영케이블은 케이블 제조 중소기업.

① 사내 운영 시스템 구축

사장님 머릿속에만 있던 생산 흐름을 시스템에 옮기는 6주 패키지. 사내 직원이 직접 쓸 수 있도록 학습까지 포함.

② AI 코칭 (월정액)

주 1회 세션 + 수시 자문 형태의 지속 운영 파트너십. 시스템 깔고 떠나는 게 아니라 같이 진화시키는 모델.

핵심: "제가 이미 회사를 AI로 운영하고 있다"가 가장 강력한 영업 증거입니다.

⑥ 미디어

회사가 자체 매체를 가지면

Sociai (사회연대경제 매체)

친환경·협동조합·복지·사회적기업은 주류 신문에 거의 안 실립니다. 그래서 직접 매체를 만듭니다.

유사하게 어반정글도 자체 매체·쇼핑몰을 준비 중.

왜 자체 매체인가

광고비는 사라지지만, 자체 매체는 회사 자산으로 쌓입니다.

"가상 마케팅 팀"

서버 맥에서 돌아가는 AI 마케팅 팀. 단톡방 인사이트 + 뉴스 자동 수집 → AI 1차 분류 → 글 초안 → 사람 검토 → 발행

관련: 카톡 분석 · 뉴스 자동 수집 · 원자재 시황 자동 브리핑

시스템화 패턴

모든 운영 항목의 4단 진화

1. 손매번 즉흥적으로
처리. 머릿속에만.
2. 메모워드·노션·카톡
적기 시작. 패턴 보임.
3. 워크플로우일부 자동화 도구.
반은 수동.
4. AI 처리AI가 받아 처리.
인간은 결정만.
중요한 건 "4단까지 가는 것"이 아니라 "3단의 항목 수를 늘리는 것"입니다.
한 항목을 끝판왕까지 끌고 가느라 시간 쓰는 것보다, 10개 항목을 2~3단에 올리는 것이 ROI가 큽니다.

하네스

"안 멈추는 시스템"이 진짜 ROI

"내가 매번 들여다봐야 하는 시스템" = 직원 한 명 더 쓰는 것과 같음.

자동화의 진짜 목표는 "AI가 일하는 것"이 아니라 "내가 안 들여다봐도 되는 것".

지난 1주일 적용

문제 ①

쇼가든 설계 — 어디까지 자동화 가능?

지금 막힌 곳

"클라이언트 인터뷰 → 도면 1차안"까지의 자동화. 식물 선정·재고 통합은 가능하나, "이 클라이언트가 진짜 원하는 분위기"를 텍스트에서 추출해 그림으로 만드는 부분이 막힘.

제약

도면은 실제 시공 가능한 식물 조합이어야 함. 계절·일조량·공간 크기 다 맞아야 함.

듣고 싶은 것

  • "분위기·취향" 같은 모호한 말을 구조화하는 패턴
  • 현실 제약(식물 크기·일조량)을 AI 생성에 강제하는 방법
  • 유사 도메인 — 인테리어·패션 자동화 사례

문제 ②

한 시스템으로 여러 회사 운영 — 안전하게

현재 상황

두 회사(어반정글·네이처커뮤니티) + 컨설팅 클라이언트 = 최소 4개 운영 컨텍스트를 한 시스템에서 굴림.

위험
